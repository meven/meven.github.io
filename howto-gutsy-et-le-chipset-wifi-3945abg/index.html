HowTo gutsy et le chipset wifi 3945ABG

2007-12-02  2 minute read

Je possède un dell xps m1330 avec une kubuntu gutsy comme système. J'ainremarqué comme d'autre que le pilote wifi plantait parfois en particuliernlorsque l'on téléchargeait de gros fichier. J'ai trouvé grâce au forum US denubuntu une solution efficace : Utiliser le pilote libre iwl3945 plutôt que lennon libre ipw3945.
nDisclaimer : attention ce que vous faites, jenne peux être tenu des conséquences éventuelles.
n

tester afin d'installer

nIl est possible de tester si votre matériel supportera bien l'opération ennremplaçant dynamiquement le module du noyau incriminé. Les changements ne sontndonc pas permanents, après un redémarrage tout sera configurer commenoriginalement.n
    n
  • d'abord arrêter le réseau # remplacé eth1 par l'interface réseau quincorrespond au wifi par exemple wlan0 si nécessaire
    nsudo ifdown eth1
    • n
  • On décharge le module défaillant sudo modprobe -r ipw3945
    • n
  • On met à plat les dépendances du module sudo depmod -a
    • n
  • On charge l'autre module sudo modprobe iwl3945
    • n
n

Installation permanente

nCe qui suit permet de se passer de ipw3945 de façon permanente (mais pasnirréversible)n
    n
  • blacklister le module ipw3945 pour empêcher le noyau de le chargernsudo bash -c 'echo "blacklist ipw3945" >n/etc/modprobe.d/blacklist-ipw3945'
    • n
  • Ajouter iwl3945 aux modules charger au démarrage sudo bash -c 'echon"iwl3945" >> /etc/modules'
    • n
  • Il faut ensuite modifier le fichiern/etc/udev/rules.d/70-persistent-net.rules pour corriger un problème vis à visndu nommage de l'interface réseau. Il faut commenter la ligne de votre ancienneninterface réseau (pour moi eth1). Vous pouvez le faire avec gedit ou bien avecnla commande : sudo sed -i "s/(.*eth1)/#1/"n/etc/udev/rules.d/70-persistent-net.rules
    • n
  • redémarrer et profiter d'une connection wifi bien plus stable !
    • n
n
